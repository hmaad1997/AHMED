# 📚 كيفية إضافة القراء والملفات الصوتية

## 🎯 نظرة سريعة

لإضافة قراء جدد للنظام، تحتاج:
1. ✅ **ملفات فيديو أو صوت** للقراء
2. ✅ **معلومات القراء** (اسم، بلد، إلخ)
3. ✅ **تشغيل سكريبت المعالجة**

---

## 📁 الخطوة 1: إضافة الملفات الصوتية/الفيديو

### 🎬 المكان:
ضع جميع الملفات في:
```
backend/data/raw_videos/
```

### 📋 أنواع الملفات المدعومة:

#### فيديو (موصى به):
- ✅ `.mp4` (الأفضل)
- ✅ `.avi`
- ✅ `.mkv`
- ✅ `.mov`
- ✅ `.flv`
- ✅ `.wmv`

#### صوت (يعمل أيضاً):
- ✅ `.wav`
- ✅ `.mp3`
- ✅ `.m4a`
- ✅ `.flac`
- ✅ `.ogg`

### 📝 تسمية الملفات:

**مهم جداً!** اسم الملف = اسم القارئ

**أمثلة:**
```
عبد_الباسط_عبد_الصمد.mp4
محمد_صديق_المنشاوي.mp4
ماهر_المعيقلي.mp4
قارئ_جديد_1.mp4
```

**قواعد التسمية:**
- ✅ استخدم `_` بدلاً من المسافات
- ✅ استخدم الأحرف العربية أو الإنجليزية
- ✅ تجنب الرموز الخاصة: `@ # $ % & *`
- ✅ لا تستخدم مسافات في الاسم

---

## 🎵 متطلبات الملفات الصوتية

### ✅ المدة:
- **الحد الأدنى:** 2-3 دقائق
- **الموصى به:** 5-15 دقيقة
- **المثالي:** 10-20 دقيقة

**لماذا؟**
- مدة أطول = دقة أعلى
- النظام يقسم الملف لمقاطع 30 ثانية
- كل مقطع يولد embedding
- المتوسط يعطي "بصمة صوتية" أدق

### ✅ الجودة:
- **صوت واضح** (بدون ضوضاء)
- **بدون موسيقى** في الخلفية
- **بدون صدى** أو echo
- **تلاوة فقط** (ليس خطبة أو محاضرة)

### ✅ المحتوى:
- **تلاوة قرآنية** فقط
- **صوت القارئ المطلوب** فقط (بدون أصوات أخرى)
- **يفضل:** مقاطع متنوعة من السور

## 🔧 الخطوة 2: معالجة البيانات

بعد وضع الملفات، شغّل سكريبت المعالجة:

```bash
cd backend
python scripts/process_data.py
```

### ⏱️ ما سيحدث:

```
1. قراءة جميع الملفات من raw_videos/
   ↓
2. استخراج الصوت (إذا كان فيديو)
   ↓
3. تقسيم الصوت لمقاطع 30 ثانية
   ↓
4. توليد Voice Embedding لكل مقطع
   ↓
5. حساب متوسط الـ embeddings
   ↓
6. حفظ "البصمة الصوتية" في قاعدة البيانات
```

### ⏳ الوقت المتوقع:
- **ملف واحد (10 دقائق):** ~2-3 دقائق
- **10 ملفات:** ~20-30 دقيقة
- **50 ملف:** ~1.5-2.5 ساعة

## 📝 الخطوة 3: إضافة معلومات القارئ (اختياري)

لإضافة معلومات القارئ (صورة، بلد، سيرة)، عدّل الملف:

```bash
backend/data/reciters_metadata.json
```

### 📄 إضافة قارئ جديد:

افتح الملف وأضف:

```json
{
  "id": 51,
  "name": "اسم القارئ بالعربي",
  "name_english": "Reciter Name in English",
  "country": "البلد",
  "bio": "نبذة عن القارئ...",
  "birth_year": "1980",
  "death_year": null,
  "image_url": "https://example.com/image.jpg",
  "recitation_style": "مرتل"
}
```

### 📋 مثال كامل:

```json
{
  "version": "1.0",
  "reciters": [
    {
      "id": 1,
      "name": "عبد الباسط عبد الصمد",
      "name_english": "Abdul Basit Abdus Samad",
      "country": "مصر",
      "bio": "قارئ مصري شهير...",
      "birth_year": "1927",
      "death_year": "1988",
      "image_url": "https://example.com/abdul_basit.jpg",
      "recitation_style": "مجود"
    },
    {
      "id": 51,
      "name": "قارئ جديد",
      "name_english": "New Reciter",
      "country": "السعودية",
      "bio": "قارئ سعودي موهوب...",
      "birth_year": "1990",
      "death_year": null,
      "image_url": "",
      "recitation_style": "مرتل"
    }
  ]
}
```

### 📌 الحقول:

| الحقل | مطلوب؟ | الوصف |
|------|--------|-------|
| `id` | ✅ نعم | رقم فريد |
| `name` | ✅ نعم | الاسم بالعربي |
| `name_english` | ✅ نعم | الاسم بالإنجليزي |
| `country` | ✅ نعم | البلد |
| `bio` | ✅ نعم | نبذة مختصرة |
| `birth_year` | ✅ نعم | سنة الميلاد |
| `death_year` | ❌ لا | سنة الوفاة (أو `null`) |
| `image_url` | ❌ لا | رابط الصورة (أو `""`) |
| `recitation_style` | ✅ نعم | نمط التلاوة (مجود/مرتل) |

## 🔄 الخطوة 4: إعادة تشغيل السيرفر

بعد إضافة القراء:


1. سجّل صوت من القارئ الجديد
2. اضغط "إيقاف وتحليل"

## 🎯 نصائح مهمة

### ✅ للحصول على أفضل النتائج:

1. **استخدم فيديوهات عالية الجودة**
   - صوت HD أفضل من SD
   - تجنب الملفات المضغوطة جداً

2. **مدة أطول = دقة أعلى**
   - 5 دقائق: جيد
   - 10 دقائق: ممتاز
   - 15+ دقيقة: مثالي

3. **صوت نقي**
   - بدون موسيقى
   - بدون ضوضاء خلفية
   - بدون صدى

4. **تنوع في المقاطع**
   - من سور مختلفة أفضل
   - أساليب تلاوة متنوعة

## ⚠️ المشاكل الشائعة وحلولها

### ❌ المشكلة: "FFmpeg not found"
```bash
# الحل:
# Windows: حمّل FFmpeg من ffmpeg.org
# Linux: sudo apt install ffmpeg
# Mac: brew install ffmpeg
```

### ❌ المشكلة: "No valid embeddings generated"
```
السبب المحتمل:
- الملف الصوتي قصير جداً (< 2 دقيقة)
- جودة الصوت سيئة جداً
- الملف تالف

الحل:
- استخدم ملف أطول وأوضح
```

### ❌ المشكلة: "Out of memory"
```
السبب:
- ذاكرة RAM غير كافية

الحل:
- أغلق البرامج الأخرى
- عالج الملفات على دفعات (10-20 ملف في المرة)
```

### ❌ المشكلة: القارئ لا يظهر في التطبيق
```
الحل:
1. تأكد من نجاح المعالجة
2. تحقق من reciter_database.json
3. أعد تشغيل Backend Server
4. أعد تشغيل التطبيق
```
